### ۱. استراتژی LLM: API-Only ✅
**تاریخ**: 2026-07-05  
**وضعیت**: تصویب‌شده

**تصمیم**: 
پروژه کاملاً API-Only خواهد بود. هیچ LLM لوکالی (مانند Ollama) پشتیبانی نمی‌شود.

**دلایل**:
- کاهش پیچیدگی کد و نگهداری
- عملکرد بهتر و پایدارتر
- کاربران خودشان API key می‌آورند (مثل Cursor)
- عدم نیاز به مدیریت مدل‌های لوکال
- مقیاس‌پذیری بهتر

**پیامدها**:
- حذف کامل Ollama از کدbase
- تمرکز بر روی abstraction layer برای APIها
- کاربران مسئول هزینه‌های API خود هستند

### ۳. Claude API به عنوان پیش‌فرض ✅
**تاریخ**: 2026-07-05  
**وضعیت**: تصویب‌شده

**تصمیم**: 
Claude API (Anthropic) به عنوان provider پیش‌فرض انتخاب شد.

**دلایل**:
- کیفیت بالای پاسخ‌ها
- Context window بزرگ
- Function Calling قوی
- پایداری API

**Providers پشتیبانی‌شده**:
1. Claude API (پیش‌فرض)
2. OpenAI API (GPT-4, GPT-3.5)
3. OpenRouter (دسترسی به چندین مدل)
4. DeepSeek API (جایگزین ارزان)

### ۴. استفاده از Transactions برای ایمنی ✅
**تاریخ**: 2026-07-05  
**وضعیت**: تصویب‌شده

**تصمیم**: 
تمام عملیات‌های تغییر مدل باید داخل Transaction اجرا شوند.

**مکانیزم**:
1. قبل از اجرا Transaction باز می‌شود
2. کد اجرا می‌شود
3. در صورت موفقیت: Commit
4. در صورت خطا: Abort

**دلایل**:
- ایمنی کامل مدل کاربر
- قابلیت بازگشت خودکار
- جلوگیری از corruption
